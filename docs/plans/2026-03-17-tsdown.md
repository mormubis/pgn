# tsdown Build Migration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to
> implement this plan task-by-task.

**Goal:** Replace `tsc` as the build tool with `tsdown` to bundle and minify the
output into a single `dist/index.js`, eliminating the separate `grammar.cjs.map`
(84KB) and shrinking the published package from ~250KB to ~60-70KB.

**Architecture:** Install `tsdown`, add `tsdown.config.ts`, replace
`build:entry` script, remove `tsconfig.build.json`. Grammar compilation
(`peggy`) is unchanged. Tests use `src/` directly and are unaffected.

**Tech Stack:** tsdown, Rolldown, pnpm

---

### Task 1: Install tsdown

**Files:**

- Modify: `package.json`

**Step 1: Install tsdown as a devDependency**

```bash
pnpm add -D tsdown
```

**Step 2: Verify installation**

```bash
./node_modules/.bin/tsdown --version
```

Expected: prints a version number (e.g. `0.16.x`).

**Step 3: Run existing tests to confirm nothing broke**

```bash
pnpm test
```

Expected: 179 tests pass.

**Step 4: Commit**

```bash
git add package.json pnpm-lock.yaml
git commit -m "chore: install tsdown"
```

---

### Task 2: Create `tsdown.config.ts`

**Files:**

- Create: `tsdown.config.ts`

**Step 1: Create the config file at the project root**

```typescript
import { defineConfig } from 'tsdown';

export default defineConfig({
  dts: true,
  entry: ['src/index.ts'],
  format: 'esm',
  minify: true,
  platform: 'neutral',
  sourcemap: true,
});
```

**Step 2: Run tsdown manually to verify it produces output**

```bash
./node_modules/.bin/tsdown
```

Expected: `dist/index.js`, `dist/index.js.map`, `dist/index.d.ts` are created.
No `dist/grammar.cjs.map`.

Note: This step will likely fail or produce incorrect output because
`src/grammar.cjs` does not exist yet — it is generated by
`pnpm grammar:compile`. Run that first:

```bash
pnpm grammar:compile && ./node_modules/.bin/tsdown
```

**Step 3: Verify key outputs**

```bash
ls dist/
```

Expected output contains:

- `index.js` — minified bundle
- `index.js.map` — source map
- `index.d.ts` — type declarations

Must NOT contain:

- `grammar.cjs`
- `grammar.cjs.map`
- `parse.js`, `stream.js`, `stringify.js`, `types.js` (these are now inlined)

**Step 4: Verify the bundle imports correctly (smoke test)**

```bash
node --input-type=module << 'EOF'
import parse from './dist/index.js';
const games = parse('[Result "1-0"]\n\n1. e4 1-0');
console.log(games.length === 1 ? 'OK' : 'FAIL');
EOF
```

Expected: prints `OK`.

**Step 5: Do NOT commit yet** — wait for Task 3 to update scripts first.

### Task 3: Update `package.json` scripts

**Files:**

- Modify: `package.json`

**Step 1: Replace `build:entry` script**

Change:

```json
"build:entry": "tsc --project tsconfig.build.json"
```

To:

```json
"build:entry": "tsdown"
```

**Step 2: Run the full build via the existing `build` script**

```bash
pnpm build
```

Expected: grammar compiles first (`pnpm grammar:compile`), then `tsdown` runs,
`dist/` is populated correctly.

**Step 3: Run all tests**

```bash
pnpm test
```

Expected: 179 tests pass. (Tests import from `src/` via Vitest, not `dist/`, so
they are unaffected by the build tool change.)

**Step 4: Run lint**

```bash
pnpm lint:ci
```

Expected: clean. Note: `tsdown.config.ts` at the project root may trigger
ESLint's glob pattern `"src/**/*.{ts,tsx}"`. If it does, either add it to the
ESLint include or add a `// eslint-disable` comment. Check `eslint.config.mjs`
to confirm whether root-level `.ts` files are linted.

**Step 5: Commit**

```bash
git add tsdown.config.ts package.json
git commit -m "feat: replace tsc build with tsdown for bundled minified output"
```

### Task 4: Remove `tsconfig.build.json`

**Files:**

- Delete: `tsconfig.build.json` (by overwriting with a redirect to tsdown)

Note: Since shell `rm` is not available, rename the file contents to a tombstone
comment, or use git to remove it:

```bash
git rm tsconfig.build.json
git commit -m "chore: remove tsconfig.build.json (superseded by tsdown)"
```

If `git rm` is not available, overwrite the file with a note and commit:

```typescript
// This file is no longer used. Build is handled by tsdown.config.ts.
// See docs/plans/2026-03-17-tsdown-design.md

```

Wait — `tsconfig.build.json` is JSON not TypeScript. If it cannot be deleted,
leave it in place — it is harmless since no script references it anymore. Skip
this task if deletion is not possible.
